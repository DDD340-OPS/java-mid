# 个人记账工具思路整理

## 1. 项目定位

本项目是一个基于 Java 控制台的个人记账工具，用于记录日常收入与支出，并生成统计报表。

这个题目适合做成“命令行 + 面向对象分层”的形式，原因有两个：

1. 实现难度适中，容易在期中作业周期内完成。
2. 能比较自然地展示类设计、集合框架、日期 API、字符串处理和统计逻辑。

## 2. 功能与题目要求对应

### 必做功能

- 创建账本
- 添加记录
- 查看记录
- 修改记录
- 删除记录
- 统计报表：总收入、总支出、结余
- 分类统计：各分类支出占比
- 月度统计：查看某月收支情况
- 数据筛选：按日期范围、分类、类型筛选

### 可展示的亮点

- 使用 `enum` 表示收入/支出类型，避免直接写魔法值
- 使用 `List` 存储交易记录，使用 `Map` 汇总分类统计
- 使用 `LocalDate` 和 `YearMonth` 完成日期筛选与月份统计
- 使用 `Stream API` 完成汇总、筛选和排序

## 3. 推荐类设计

### 核心实体类

- `Transaction`
  - 表示一条交易记录
  - 字段：`id`、`date`、`type`、`category`、`amount`、`note`

- `Ledger`
  - 表示一个账本
  - 字段：`name`、`transactions`
  - 职责：保存和管理所有交易记录

- `TransactionType`
  - 枚举类
  - 值：`INCOME`、`EXPENSE`

### 业务类

- `LedgerService`
  - 负责添加、修改、删除、查询记录

- `StatisticsService`
  - 负责统计总收入、总支出、结余
  - 负责分类统计和月度统计

- `FilterService`
  - 负责按日期范围、分类、类型进行筛选

### 工具类

- `DateUtil`
  - 日期解析与格式化

- `IdGenerator`
  - 生成记录 ID

- `InputHelper`
  - 统一处理命令行输入和校验

### 交互层

- `LedgerConsoleUI`
  - 展示菜单
  - 接收用户输入
  - 调用业务层完成功能

- `LedgerApp`
  - 程序入口

## 5. 可以讲的面向对象设计点

### 封装

每个类只负责自己的事情，例如：

- `Transaction` 只负责一条记录的数据
- `Ledger` 只负责账本和记录集合
- `StatisticsService` 只负责统计

### 单一职责

避免把所有代码都写进 `main` 方法。这样结构清晰，也更符合老师对“面向对象设计”的评分要求。

### 扩展性

如果后续要继续扩展，可以很自然地增加：

- 数据持久化
- 报表导出
- 图形界面
- 多账本管理

## 6. 汇报时推荐展示的 4 个代码亮点

### 亮点 1：交易类型枚举

把收入/支出做成枚举，不直接写 `1`、`2`，更规范，也更安全。

### 亮点 2：统计报表逻辑

使用 `Stream API` 分别统计总收入与总支出，再算出结余。

### 亮点 3：分类统计

使用 `Map<String, Double>` 汇总每种分类的支出金额，再计算占比。

### 亮点 4：月度筛选

使用 `YearMonth` 判断记录是否属于目标月份，代码比旧版 `Date` 处理方式更清晰。

## 7. 现场演示建议流程

### 演示路线

1. 创建账本
2. 添加几条收入和支出记录
3. 查看全部记录
4. 修改其中一条错误记录
5. 删除一条测试记录
6. 生成统计报表
7. 查看分类统计
8. 查看某月统计
9. 按分类和时间范围筛选数据

### 演示时可以强调

- 添加一条收入记录和一条支出记录
- 展示“按日期排序”
- 展示“分类支出占比”
- 展示“月度统计”
- 展示“筛选结果”

在项目目录下执行：

```bash

```

